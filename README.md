# 布丁猫 Ozon ERP - 电商管理系统

一个功能完善的 Ozon 电商平台管理后台，参考布丁猫 ERP 设计，支持店铺管理、快速上货、采集上货、订单管理、库存管理、仓库管理、活动管理等功能。

## 🚀 快速开始

### 安装依赖
```bash
npm install
```

### 启动开发服务器
```bash
npm run dev
```

### 构建生产版本
```bash
npm run build
```

### 部署到 GitHub Pages（推荐）
```
双击 deploy.bat
```
会自动完成：构建 → 把 dist 铺平到 gh-pages 分支 → 强制推送 → 1-2 分钟后访问 https://cqx1028.github.io/qxERP/

> GitHub Pages 配置：Settings → Pages → Branch: `gh-pages` / (root)。如果 Pages 显示 404，请重新点一次 Save。
>
> 脚本会调用 `deploy.cjs`（Node 实现），需要先装 Node.js。Git 推送时如弹出 GitHub 登录窗口，输入 PAT 作为密码即可。

## ✅ 真实 API 数据覆盖范围（2026-07-23 实测）

| 模块 | API 端点 | 数据 |
|------|----------|------|
| 店铺信息 | `/v1/seller/info` | 公司名/INN/评级/Premium |
| 订单（FBO） | `/v2/posting/fbo/list` | 全部 FBO 订单 |
| 商品列表 | `/v3/product/list` | 全部商品 ID/SKU/存档状态 |
| 商品详情 | `/v3/product/info/list` | 名称/价格/图片/佣金/状态 |
| 商品库存 | `/v4/product/info/stocks` | FBS 库存数量 |
| 仓库列表 | `/v2/warehouse/list` | 仓库名称/城市/类型 |
| 销售分析 | `/v1/analytics/data` | 30天 SKU 维度收入/销量 |
| 类目树 | `/v1/description-category/tree` | 26 个类目及子类 |
| 财务流水 | `/v3/finance/transaction/list` | 按 posting_number 查询 |
| 评价 | `/v1/review/list` | 免费版无权限 |

> **注意**：评价接口（`/v1/review/list`）在免费版 Ozon 账户下返回 PermissionDenied，需付费版。
> 财务流水接口需指定 `posting_number` 或完整的 `date_from`/`date_to` 参数。
│   └── main.jsx         # 入口文件
├── dist/                # 生产构建输出
├── index.html           # HTML 模板
├── tailwind.config.js   # Tailwind 配置
├── postcss.config.js    # PostCSS 配置
├── vite.config.js       # Vite 配置
├── start.bat            # Windows 启动脚本
├── build.bat            # Windows 构建脚本
├── deploy.bat           # Windows 部署脚本（双击推到 GitHub Pages）
├── deploy.cjs           # 部署脚本实现（避免 cmd 编码坑）
└── package.json         # 项目依赖
```

### 🕷️ 采集上货 · 一键跟卖（真实货源解析管道）
- **真实解析**：粘贴商品链接或关键词后，由「服务端代理」(`/api/crawl`，见 `server/parser.mjs`) 在服务端真实抓取货源页（1688/淘宝/拼多多/京东），解析标题/价格/图片/规格，再回传前端预览。浏览器端无法直连这些站点（跨域 + 反爬），因此必须由服务端代理。
- 解析成功显示「真实解析」徽标；若货源站返回内容不足（反爬/JS 渲染）则回退「演示数据」并注明原因。
- 「导入到商品库」后可在「商品管理」编辑上架，记录进入「采集上传记录」。
- 说明：货源站（尤其京东/淘宝详情页）多为 JS 渲染 + 反爬，原始 HTML 常只返回壳页；直连商品链接成功率高于关键词搜索。要全平台稳定解析，生产环境建议叠加无头浏览器或浏览器扩展（cookie 注入，同毛子ERP思路）。

## 💾 数据说明

### 数据存储
- 所有数据保存在浏览器 localStorage 中
- 刷新页面数据不会丢失
- 清除浏览器数据会丢失所有记录

### 模拟数据（默认）
- 订单：48条模拟订单
- 商品：10个示例商品
- 仓库：3个示例仓库
- 采集记录：20条示例记录
- 活动：3个示例活动
- 消息：4条示例消息

### 真实数据（绑定 API 后）
- 在「系统设置」填入 Ozon 的 Client-Id 与 Api-Key 并测试连接成功后，系统自动从 Ozon 卖家 API 拉取真实数据
- 拉取范围：订单（FBS/FBO）、商品（含库存与价格）、仓库、**评价（reviews）**、**财务交易（transactions）**
- 订单/商品/库存/仓库页面及仪表盘统计均展示真实数据；**消息提醒**页会并入真实评价；接入失败时回退模拟数据
- 仪表盘「订单数量」趋势图按真实订单日期聚合
- 已预留接口：销售分析（/v2/analytics/data）、品类佣金（/v1/commission/tariff），可在「数据分析/财务」页接入
